# Contributing Guidelines

## Purpose

Explain how contributors should work in a documentation-first repository.

## Principles

- Documentation comes before implementation.
- Every significant change should reference a related document or decision.
- Keep changes small enough to review independently.
- Prefer clarity over brevity when the choice affects correctness.

## Workflow

1. Identify the change.
2. Update or create the relevant documentation.
3. Record lasting decisions in an ADR or decision log.
4. Review for traceability, naming, and link integrity.
5. Approve before implementation work begins.

## Review Expectations

- State the purpose of the change.
- List dependencies and related docs.
- Call out risks, assumptions, and follow-up items.
- Confirm the document is self-contained enough for review.
