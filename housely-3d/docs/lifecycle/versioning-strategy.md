# Versioning Strategy

## Purpose

Define how the project version communicates maturity, compatibility, and release cadence.

## Rules

- Use a consistent versioning scheme across the repository and release notes.
- Treat breaking changes as first-class events that require explicit documentation.
- Record the rationale for non-obvious version jumps.

## Dependencies

- Release strategy.
- Backup strategy.
- Decision logs.

## Review Checklist

- Is the versioning scheme stable.
- Can users understand change impact from the version number and notes.
- Are breaking changes visible.
