# Development Lifecycle

## Purpose

Describe the end-to-end path from documentation to implementation, review, release, and post-release learning.

## Lifecycle Stages

- Discovery and documentation.
- Design review and decision capture.
- Implementation.
- Verification and QA.
- Release readiness.
- Production monitoring.
- Retrospective and follow-up documentation.

## Dependencies

- Git strategy.
- Branching strategy.
- Testing structure.
- Release strategy.

## Review Checklist

- Is the stage order clear.
- Are handoff criteria explicit.
- Are documentation updates required at each transition.
