# Branching Strategy

## Purpose

Describe how branches are organized so work stays reviewable and releasable.

## Rules

- Use short-lived branches for scoped work.
- Use one branch per major objective.
- Protect the mainline branch.
- Keep documentation changes readable and isolated where possible.

## Dependencies

- Git strategy.
- Release strategy.

## Review Checklist

- Is branch intent obvious.
- Are merge boundaries clean.
- Does the strategy support release confidence.
