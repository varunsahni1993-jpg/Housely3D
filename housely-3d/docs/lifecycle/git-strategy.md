# Git Strategy

## Purpose

Define how Git is used to keep changes auditable and safe.

## Rules

- Keep commits focused on one logical change.
- Link commits and pull requests to the related document or decision.
- Avoid mixed-purpose branches.
- Preserve historical context for decisions and releases.

## Dependencies

- Branching strategy.
- Versioning strategy.
- Decision log template.

## Review Checklist

- Is the change easy to trace.
- Is the intent visible from the commit or PR trail.
- Does the Git history support auditability.
