# Application Backlog

## Purpose

Track intentionally deferred work so the shell stays focused while the product roadmap remains visible.

## Deferred Work

- Authentication.
- Project CRUD.
- Three.js viewer.
- Asset library.
- AI services.
- Backend integration.
- Notifications.
- Command palette.
- Internationalization.
- Offline support.

## Usage Rules

- Keep backlog items visible but out of implementation scope until they are formally promoted.
- Move an item into a phase document or ADR before starting implementation.
- Update the backlog when scope changes or when a feature is intentionally deferred.
