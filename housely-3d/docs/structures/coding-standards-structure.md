# Coding Standards Structure

## Purpose

Explain how coding standards should be organized so implementation rules remain stable and easy to audit.

## Structure

- Language-agnostic standards.
- Frontend standards.
- Backend standards.
- AI and prompt integration standards.
- Testing and review standards.

## Dependencies

- Documentation standards.
- Development lifecycle.
- Security documentation structure.

## Review Checklist

- Is the structure broad enough for future services.
- Does it separate stable rules from implementation details.
- Can teams find the rule they need quickly.
