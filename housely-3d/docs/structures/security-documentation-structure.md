# Security Documentation Structure

## Purpose

Define the security documentation stack for protecting data, prompts, models, services, and user workflows.

## Structure

- Security principles and policy.
- Threat model and trust boundaries.
- Secrets handling and access control.
- Data classification and retention.
- Incident response and recovery.

## Dependencies

- Architecture documentation plan.
- Backup strategy.
- Release strategy.

## Review Checklist

- Are trust boundaries clear.
- Are sensitive assets named.
- Are responsibilities explicit.
- Can a reviewer understand the controls without code.
