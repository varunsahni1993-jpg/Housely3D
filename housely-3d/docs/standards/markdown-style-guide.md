# Markdown Style Guide

## Purpose

Keep documentation clean, consistent, and easy to scan.

## Rules

- Use short headings with one idea per section.
- Prefer bullet lists for structure and tables for comparisons.
- Keep paragraphs short.
- Use relative links within the repository.
- Use code fences only for templates, commands, or structured examples.
- Avoid decorative formatting that does not add meaning.

## Writing Pattern

- Purpose
- Scope
- Decisions
- Dependencies
- Risks
- Related
