# Folder Responsibilities

## Purpose

Define what each repository folder is responsible for so documents do not drift into the wrong location.

## Top-Level Folders

- `book/` - engineering handbook volumes and chapter skeletons.
- `docs/` - master documentation system, standards, plans, and structures.
- `prompts/` - prompt library governance and prompt-family documentation.
- `research/` - research notes, experiments, and source summaries.
- `templates/` - reusable templates for all documentation types.
- `decisions/` - ADRs, decision logs, and decision history.
- `backups/` - backup policy references and recovery guidance.

## Recommended Subfolders

- `book/volumes/` - volume folders grouped by topic.
- `docs/standards/` - documentation standards and writing rules.
- `docs/plans/` - documentation plans for major functional areas.
- `docs/lifecycle/` - Git, versioning, release, and recovery policies.
- `docs/structures/` - structure docs for security, testing, prompts, and research.
- `prompts/library/` - reusable prompt families and canonical prompt records.
- `research/sources/` - source-backed research records.
- `research/experiments/` - experiment notes and outcomes.
