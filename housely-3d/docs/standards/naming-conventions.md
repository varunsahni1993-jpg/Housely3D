# Naming Conventions

## Purpose

Create a predictable naming system for files, folders, and documents.

## Rules

- Use lowercase kebab-case for markdown filenames.
- Prefer descriptive names over short abbreviations.
- Keep decision identifiers stable and monotonic, such as `ADR-0001`.
- Use numbered prefixes for ordered sequences only when order matters.
- Use singular folder names for document categories unless the category is naturally plural.

## Examples

- `architecture-documentation-plan.md`
- `branching-strategy.md`
- `research-template.md`
- `ADR-0001-platform-boundaries.md`
