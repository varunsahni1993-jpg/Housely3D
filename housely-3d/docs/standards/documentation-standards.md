# Documentation Standards

## Purpose

Define the rules that keep documentation production-ready, reviewable, and durable over a multi-year product lifecycle.

## Standards

- Documentation is the source of truth for scope, architecture, and operating rules.
- Every document must state its purpose and link related documents.
- Every change that affects direction must create or update an ADR or decision log entry.
- Every structural document must explain why the structure exists, not only what it contains.
- Every phase document must be independently reviewable.

## Review Gates

- Clarity: can a new contributor understand the intent quickly.
- Traceability: are related docs and decisions linked.
- Durability: is the guidance stable enough for long-term use.
- Reviewability: can this document be approved on its own.
