# Repository Overview

## Purpose

Provide a concise map of the repository so contributors understand where each class of documentation belongs.

## Top-Level Areas

- `book/` - engineering handbook and company constitution.
- `docs/` - master documentation system, standards, plans, lifecycle, and structures.
- `decisions/` - ADRs and decision logs.
- `prompts/` - prompt library governance and prompt families.
- `research/` - sources, experiments, findings, and notes.
- `templates/` - reusable document templates.
- `backups/` - backup policy references and recovery notes.
